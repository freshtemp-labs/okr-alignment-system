## 1. 项目概述

### 1.1 背景

OKR（Objectives and Key Results，目标与关键结果）是现代企业广泛采用的目标管理方法论。本系统旨在解决企业 OKR 管理中"上下对齐难、进度追踪难、可视化难"三大痛点，提供一套覆盖企业级到个人级的级联对齐管理工具。

基于现有 HTML Demo 原型，需要将功能扩展为完整的跨平台原生应用（macOS + iOS），支持从企业战略目标的制定、分解、对齐，到个人 OKR 的执行、追踪、反馈的全生命周期管理。

### 1.2 产品愿景

> 打造一款优雅、直观、高效的 OKR 对齐管理工具，让组织目标清晰可见，让个人贡献可被衡量，让级联对齐自然发生。

### 1.3 目标用户

| 用户角色 | 描述 | 核心诉求 |
|----------|------|----------|
| **企业管理者 / CEO** | 制定公司级战略目标，关注整体达成情况 | 全局视角、战略落地、组织健康度 |
| **部门负责人 / 总监** | 将企业 OKR 分解为部门/团队 OKR | 级联分解、团队绩效、资源协调 |
| **团队管理者 / 项目经理** | 管理团队 OKR 的落地执行 | 进度追踪、风险预警、成员协作 |
| **普通员工** | 制定个人 OKR 并对齐到上级目标 | 目标清晰、进度反馈、自我驱动 |
| **HR / OKR 管理员** | 推动 OKR 制度落地，组织复盘 | 流程管理、数据报表、最佳实践 |

### 1.4 产品范围

**包含范围：**
- 企业级与个人级 OKR 的 CRUD 管理
- 树状可视化对齐视图（核心差异化功能）
- 级联进度自动计算与追踪
- 多用户协作与权限管理
- 跨平台数据同步（iCloud / 自建后端）
- 通知提醒与 Check-in 机制
- 数据分析与报表

**不包含范围（未来版本考虑）：**
- 与第三方项目管理工具的深度集成（Jira/飞书/钉钉等）
- AI 智能目标推荐
- 绩效考核与薪酬关联模块
- 多租户 SaaS 版本（本版本聚焦个人/团队私有化部署）

## 2. 术语与定义

| 术语 | 英文 | 定义 |
|------|------|------|
| 目标 | Objective | 定性描述"想要达成什么"，具有挑战性、鼓舞人心、可定性衡量 |
| 关键结果 | Key Result (KR) | 定量描述"如何衡量目标达成"，需具体、可量化、可检验 |
| 企业级 OKR | Enterprise OKR | 公司/组织层面的 OKR，由管理层制定 |
| 个人级 OKR | Personal OKR | 员工个人层面的 OKR，需对齐到上级目标 |
| 级联 | Cascade | OKR 从上到下逐级分解对齐的过程 |
| 对齐 | Alignment | 下级 OKR 与上级 OKR 之间的关联关系 |
| 叶子 KR | Leaf KR | 最底层无子节点的关键结果，是实际执行单元 |
| 汇总进度 | Roll-up Progress | 父节点进度由子节点进度自动汇总计算得出 |
| Check-in | Check-in | 定期更新 OKR 进度和状态的仪式 |
| 周期 | Cycle | OKR 的执行时间段，通常为季度或月度 |

### 2.1 数据模型定义

基于 Demo 原型，核心数据模型定义如下：

```
OKRNode:
  - id: String (唯一标识)
  - type: Enum ["objective" | "key_result"]
  - scope: Enum ["enterprise" | "department" | "team" | "personal"]
  - owner: User (负责人)
  - title: String (标题)
  - description: String (描述，可选)
  - currentValue: Number (当前值，仅叶子 KR)
  - targetValue: Number (目标值，仅叶子 KR)
  - unit: String (单位，如 "%", "人", "fps" 等)
  - progress: Number (进度百分比，0-100，自动计算)
  - status: Enum ["not_started" | "in_progress" | "at_risk" | "completed" | "cancelled"]
  - startDate: Date
  - dueDate: Date
  - parentId: String (父节点 ID，根节点为空)
  - children: OKRNode[] (子节点)
  - alignTo: String[] (对齐到的目标 ID)
  - tags: String[] (标签)
  - createdAt: Date
  - updatedAt: Date
```

## 3. 功能需求

### 3.1 OKR 管理模块

#### 3.1.1 OKR 生命周期管理

**FR-OKR-001 OKR 创建**
- 支持创建企业级 Objective 和 Key Result
- 支持创建个人级 Objective 和 Key Result
- 创建时支持选择父节点（建立级联关系）
- 支持设置标题、描述、负责人、起止时间
- 叶子 KR 支持设置 currentValue、targetValue、unit
- 支持批量创建（从模板导入）

**FR-OKR-002 OKR 编辑**
- 支持修改 OKR 的所有字段
- 修改叶子 KR 的 currentValue 时自动触发级联进度重算
- 支持拖拽调整节点父子关系（改变对齐关系）
- 支持修改节点 scope（企业级/个人级转换）

**FR-OKR-003 OKR 删除**
- 支持删除单个节点
- 删除父节点时支持选择：级联删除子节点 或 将子节点上移
- 删除操作需二次确认
- 支持软删除（可恢复）

**FR-OKR-004 OKR 查看**
- 支持列表视图查看（按层级、负责人、状态筛选）
- 支持树状可视化视图（核心视图，详见 3.2）
- 支持详情面板查看完整信息
- 支持查看 OKR 变更历史

#### 3.1.2 OKR 周期管理

**FR-OKR-005 周期创建与管理**
- 支持创建 OKR 周期（如 Q1 2024、Q2 2024）
- 支持设置周期起止日期
- 支持周期模板（快速创建季度/月度周期）
- 支持周期归档（历史周期只读查看）

**FR-OKR-006 周期切换**
- 支持在不同周期之间切换查看
- 支持跨周期对比（查看历史同期数据）
- 支持周期末自动提醒评估与复盘

#### 3.1.3 OKR 对齐管理

**FR-OKR-007 对齐关系建立**
- 支持将个人 O 对齐到企业 KR（建立父子关系）
- 支持多对一对齐（多个个人 O 对齐到同一个企业 KR）
- 支持可视化展示对齐路径

**FR-OKR-008 对齐健康度检测**
- 自动检测未对齐的 OKR（孤儿节点）
- 检测过度对齐（一个 OKR 对齐到过多父节点）
- 提供对齐覆盖率报表

#### 3.1.4 模板与复用

**FR-OKR-009 OKR 模板**
- 提供常用行业模板库
- 支持将已有 OKR 保存为模板
- 支持从模板快速创建新周期 OKR

**FR-OKR-010 复制与继承**
- 支持跨周期复制 OKR（新周期继承上期结构）
- 支持复制时保留或重置进度

### 3.2 树状可视化模块

#### 3.2.1 核心树状视图

**FR-VIZ-001 树状布局渲染**
- 以企业 O 为根节点，向下级联展示所有子节点
- 同一层级的节点在同一行水平排列
- 使用连接线直观展示父子关系
- 节点按类型（Objective / Key Result）和范围（Enterprise / Personal）区分样式
- 支持响应式布局（节点过多时自动缩放或启用横向滚动）

**FR-VIZ-002 节点卡片设计**
- 每个节点以卡片形式展示，包含：
  - 类型标签（Objective / Key Result）
  - 范围标识（企业级金色边框 / 个人级蓝色边框）
  - 负责人 Badge
  - 标题文本
  - 进度条（带动画效果）
  - 进度百分比数字
  - 叶子 KR 显示当前值/目标值（如 "5 / 50 人"）
  - 非叶子节点显示"对齐汇总完成度"提示
- 卡片 Hover 效果：轻微上浮 + 阴影增强

**FR-VIZ-003 连接线绘制**
- 父节点与子节点之间使用垂直+水平线条连接
- 线条颜色：半透明白色（深色主题下）
- 线条宽度：2px
- 支持动画展开/收起时的线条过渡效果

#### 3.2.2 交互功能

**FR-VIZ-004 节点展开/收起**
- 支持点击节点收起/展开其子树
- 收起状态需视觉提示（如展开图标）
- 支持一键展开/收起全部节点
- 支持记忆用户的展开/收起状态

**FR-VIZ-005 节点拖拽**
- 支持拖拽节点调整其在树中的位置
- 拖拽时实时显示可放置区域高亮
- 支持跨父节点移动（改变对齐关系）
- 拖拽结束后自动重新计算级联进度

**FR-VIZ-006 缩放与导航**
- 支持整体缩放树状视图（双指捏合 / 鼠标滚轮）
- 支持平移拖动画布
- 支持缩放到适应屏幕（Fit to Screen）
- 支持聚焦到特定节点（自动定位并高亮）

**FR-VIZ-007 节点详情浮层**
- 点击节点弹出详情侧边栏/浮层
- 展示完整信息：描述、创建时间、更新时间、历史进度
- 支持在详情中直接编辑
- 支持在详情中查看备注/评论

#### 3.2.3 视图切换

**FR-VIZ-008 多视图模式**
- 树状视图（核心视图，Demo 展示）
- 列表视图（表格形式，支持排序筛选）
- 看板视图（按状态分栏：未开始/进行中/有风险/已完成）
- 甘特图视图（时间轴展示，显示起止时间）

**FR-VIZ-009 筛选与高亮**
- 按负责人筛选（只显示某人的 OKR）
- 按状态筛选（只显示特定状态的节点）
- 按范围筛选（只显示企业级或个人级）
- 搜索高亮（输入关键词高亮匹配的节点）

### 3.3 进度追踪模块

#### 3.3.1 级联进度计算引擎

**FR-PROG-001 叶子 KR 进度计算**
- 公式：`progress = (currentValue / targetValue) x 100%`
- 结果限制在 0% - 100% 之间（clamp）
- 支持超过 100% 的显示（特殊标识，如 "120% - 超额完成"）

**FR-PROG-002 父节点进度计算**
- 公式：`progress = 所有直接子节点 progress 的平均值`
- 递归计算：从叶子节点向上逐层汇总
- 核心逻辑：企业 KR 的进度严格由其下所有个人 OKR 的完成度决定
- 计算过程实时、自动、无需手动触发

**FR-PROG-003 进度更新机制**
- 支持直接在叶子 KR 节点上更新 currentValue
- 支持 +/- 10% 快捷调整按钮（Demo 已实现）
- 支持自定义数值输入
- 支持滑动条拖拽调整
- 更新后自动触发级联重算并动画刷新进度条

#### 3.3.2 进度历史与趋势

**FR-PROG-004 进度日志**
- 每次进度变更自动记录日志
- 日志包含：旧值、新值、变更人、变更时间、备注
- 支持查看变更历史

**FR-PROG-005 进度趋势图**
- 为每个 OKR 节点展示进度变化趋势折线图
- 支持按周/月聚合查看
- 支持对比计划进度与实际进度

#### 3.3.3 Check-in 机制

**FR-PROG-006 Check-in 流程**
- 支持定期 Check-in（每周/每两周）
- Check-in 时更新进度 + 填写信心指数 + 添加备注
- 支持设置 Check-in 提醒
- Check-in 历史记录与回顾

**FR-PROG-007 信心指数**
- 支持为 OKR 设置信心指数（0-10 分）
- 信心指数独立于心进度百分比
- 信心指数变化趋势可视化

### 3.4 用户与权限模块

#### 3.4.1 用户管理

**FR-USER-001 用户账户**
- 支持 Apple ID 登录（Sign in with Apple）
- 支持邮箱+密码注册登录
- 支持企业 SSO 集成（可选）
- 支持 Guest 模式（本地使用，后续注册可迁移数据）

**FR-USER-002 用户资料**
- 支持设置头像、姓名、职位、部门
- 支持绑定多个组织/团队
- 支持个人偏好设置（主题、通知、语言）

**FR-USER-003 团队成员管理**
- 支持邀请团队成员（通过链接/邮箱）
- 支持角色分配
- 支持查看团队成员的 OKR

#### 3.4.2 权限控制

**FR-USER-004 角色定义**

| 角色 | 权限范围 |
|------|----------|
| **超级管理员** | 全部权限，包括组织设置、成员管理 |
| **企业 OKR 管理员** | 创建/编辑/删除企业级 OKR，查看全部个人 OKR |
| **部门负责人** | 创建/编辑/删除所在部门 OKR，查看部门成员 OKR |
| **普通成员** | 创建/编辑/删除自己的个人 OKR，查看相关上级 OKR |
| **访客/只读** | 仅查看被授权的 OKR，不可编辑 |

**FR-USER-005 数据隔离**
- 用户只能查看被授权的 OKR 数据
- 企业级 OKR 默认全员可见
- 个人 OKR 默认仅自己+上级可见
- 支持自定义可见范围（分享给特定人）

**FR-USER-006 审批流程**
- 个人 OKR 创建后需上级确认对齐关系
- 企业 OKR 变更需通知相关人员
- 支持审批记录查看

### 3.5 数据同步模块

#### 3.5.1 云端同步

**FR-SYNC-001 iCloud 同步**
- 支持通过 iCloud 在 macOS / iOS 间同步数据
- 使用 CloudKit 框架实现
- 支持离线编辑，网络恢复后自动同步
- 冲突自动解决策略（最后写入优先 + 冲突提示）

**FR-SYNC-002 自建后端同步（可选）**
- 支持配置企业私有服务器地址
- 使用 RESTful API 进行数据同步
- 支持端到端加密传输
- 支持团队协作模式（多用户实时同步）

#### 3.5.2 本地数据管理

**FR-SYNC-003 离线支持**
- 完整支持离线查看和编辑
- 本地 SQLite/Core Data 存储
- 离线操作的变更队列，联网后批量同步

**FR-SYNC-004 数据备份与恢复**
- 支持手动导出数据（JSON / CSV 格式）
- 支持从备份文件恢复
- 支持自动备份（保留最近 N 个版本）

**FR-SYNC-005 数据迁移**
- 支持从 Demo 数据导入
- 支持从其他 OKR 工具导入（CSV 模板）

### 3.6 通知与提醒模块

#### 3.6.1 推送通知

**FR-NOTIF-001 Check-in 提醒**
- 在设定的 Check-in 时间发送通知
- 支持自定义提醒时间
- 支持 macOS / iOS 本地通知 + APNs 远程通知

**FR-NOTIF-002 进度风险预警**
- 当 OKR 进度严重滞后时自动预警
- 预警规则可配置（如：时间过半但进度不足 30%）
- 通知发送给负责人和相关上级

**FR-NOTIF-003 变更通知**
- 当子节点进度变更影响父节点时通知父节点负责人
- 当 OKR 被编辑/删除时通知相关人员
- 支持通知频率设置（实时/摘要模式）

**FR-NOTIF-004 周期提醒**
- 周期开始提醒（制定 OKR）
- 周期中点提醒（中期回顾）
- 周期结束提醒（总结复盘）

#### 3.6.2 应用内通知中心

**FR-NOTIF-005 通知中心**
- 应用内统一通知中心查看所有通知
- 支持标记已读/全部已读
- 支持通知筛选（按类型/时间）
- 点击通知跳转到相关 OKR 页面

### 3.7 报表与分析模块

#### 3.7.1 仪表盘

**FR-REPORT-001 组织仪表盘**
- 整体 OKR 完成率（按周期）
- 对齐覆盖率（已对齐 OKR / 总 OKR）
- 平均完成度趋势图
- 风险 OKR 列表（进度严重滞后）

**FR-REPORT-002 个人仪表盘**
- 个人 OKR 完成情况概览
- 个人贡献度分析（对齐到哪些企业目标）
- 进度更新频率统计

#### 3.7.2 分析报表

**FR-REPORT-003 完成度分析**
- 按部门/团队/个人维度的完成度统计
- 按周期对比完成度变化
- 导出报表（PDF / Excel）

**FR-REPORT-004 对齐分析**
- 可视化展示对齐网络图
- 识别未对齐的孤儿 OKR
- 对齐深度统计（平均级联层数）

**FR-REPORT-005 健康度评分**
- 组织 OKR 健康度综合评分
- 多维度评分：完成率、对齐率、更新频率、信心指数
- 健康度趋势变化

## 4. 非功能需求

### 4.1 性能需求

| 需求 ID | 需求描述 | 指标 |
|---------|----------|------|
| NFR-PERF-001 | 树状视图首次渲染时间 | <= 500ms（100 个节点以内） |
| NFR-PERF-002 | 进度更新响应时间 | <= 100ms（级联重算 + UI 刷新） |
| NFR-PERF-003 | 节点拖拽流畅度 | >= 60fps 动画帧率 |

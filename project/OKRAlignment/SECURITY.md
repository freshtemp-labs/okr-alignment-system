# 安全政策 / Security Policy

## 支持版本 / Supported Versions

以下版本的 OKR Alignment System (OAMS) 目前仍在接受安全更新支持：

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 持续支持 |
| < 1.0 | ❌ 不再支持 |

## 报告漏洞 / Reporting a Vulnerability

我们非常重视项目的安全性。如果您发现了安全漏洞，请按照以下流程 responsibly disclosure（负责任的披露）：

### ⚠️ 请不要在公开的 Issue 中报告安全漏洞

公开披露安全漏洞可能会给所有用户带来风险。请通过以下私人渠道报告：

1. **发送安全报告邮件** 至项目维护团队
2. 在邮件主题中注明 `[SECURITY]` 前缀
3. 提供尽可能详细的漏洞描述，包括：
   - 漏洞类型（如：数据泄露、权限绕过、注入攻击等）
   - 受影响的版本号
   - 复现步骤
   - 可能的影响范围
   - 建议的修复方案（如有）

### 响应流程

1. **确认收到**：我们将在 48 小时内确认收到您的报告
2. **调查评估**：我们将在 7 天内完成初步调查并评估漏洞严重程度
3. **修复计划**：根据漏洞严重程度制定修复时间表：
   - **严重 (Critical)**：72 小时内发布修复
   - **高危 (High)**：14 天内发布修复
   - **中危 (Medium)**：30 天内发布修复
   - **低危 (Low)**：下次常规更新中修复
4. **披露**：修复完成后，我们将在发布说明中致谢报告者（在获得同意的情况下）

### 安全更新通知

- 安全更新将通过 GitHub Releases 发布
- 建议订阅 Releases 通知以及时获取安全更新
- 所有安全修复版本号将遵循 [SemVer](https://semver.org/lang/zh-CN/) 规范

## 安全最佳实践

在使用 OAMS 时，请遵循以下安全建议：

- 始终保持应用更新到最新版本
- 使用强密码保护您的 Apple ID 和 iCloud 账户
- 定期审查已同步到 CloudKit 的数据
- 在共享设备上启用文件保险箱 (FileVault)
- 不要在不安全的网络环境中进行敏感操作
